# 模型评估指标说明

本项目是 26 类手写英文字母分类任务，以下指标用于衡量 CNN 模型和传统模型的识别效果。

## Accuracy

准确率，表示模型预测正确的样本数占全部测试样本数的比例。

公式含义：预测正确越多，Accuracy 越高。适合类别分布比较均衡的分类任务。

## Top-5 Accuracy

Top-5 准确率，表示真实类别是否出现在模型预测概率最高的前 5 个类别中。

对于 26 个字母分类任务，如果模型第一选择不对，但真实字母在前 5 个候选中，也会被计为正确。该指标可以反映模型候选识别能力。

## Precision

精确率，表示模型预测为某一类的样本中，真正属于该类的比例。

例如模型预测为字母 A 的样本里，有多少确实是 A。Precision 越高，说明误报越少。

## Recall

召回率，表示某一类真实样本中，被模型正确识别出来的比例。

例如所有真实字母 A 中，有多少被模型识别成 A。Recall 越高，说明漏识别越少。

## F1-score

F1 分数是 Precision 和 Recall 的综合指标，用于同时衡量误报和漏报情况。

F1 越高，说明模型在精确率和召回率之间取得了更好的平衡。

## Macro Average

宏平均，表示先分别计算每个类别的指标，再对所有类别取平均。

它对每个类别一视同仁，适合观察模型是否在所有字母类别上表现均衡。

## Weighted Average

加权平均，表示按照每个类别的样本数量对指标进行加权平均。

样本数量多的类别对最终结果影响更大。若类别分布不完全均衡，该指标更接近整体测试表现。

## RMSE

均方根误差，原本常用于回归任务。本项目中它根据真实标签编号和预测标签编号之间的差异计算。

RMSE 越小，说明预测标签编号与真实标签编号整体差距越小。但由于字母编号本身不代表真实语义距离，因此本项目中 RMSE 只作为辅助参考指标，主要仍看 Accuracy、F1 和 AUC。

## ROC-AUC

ROC-AUC 表示模型区分不同类别的能力。本项目使用 OvR，也就是 One-vs-Rest 方法，将多分类任务拆成多个“某一类 vs 其他类”的二分类问题后计算。

AUC 越接近 1，说明模型越能把正确类别和其他类别区分开。

## Confusion Matrix

混淆矩阵用于观察模型把每个真实类别预测成了哪些类别。

对角线上的数值表示预测正确的数量，非对角线上的数值表示混淆错误。例如真实是 A 却预测成 B，就会出现在 A 行 B 列。

## 本项目主要参考指标

本项目是分类任务，建议重点关注：

1. Accuracy：整体识别准确率。
2. Macro F1-score：各字母类别的综合均衡表现。
3. Weighted F1-score：考虑样本数量后的整体分类表现。
4. ROC-AUC：模型区分各类别的能力。
5. Confusion Matrix：分析哪些字母容易被混淆。
